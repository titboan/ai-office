# Ева — личный секретарь: Telegram-дайджест + дайджест почты

Статус: в работе
Дата: 2026-06-13

## Контекст

Ева написана (agents/eva.py): дайджест Telegram-каналов + email-дайджест двух ящиков.
Email-часть (Фазы 2–6) реализована. Railway блокирует порт 993 (IMAP) → решение: IMAP-прокси на Fly.io.

---

## Архитектура email (итоговая)

```
Railway (eva) → HTTPS → Fly.io IMAP-прокси → порт 993 → Gmail
                                                        → Яндекс
```

Локально: `IMAP_PROXY_URL` пустой → прямое IMAP (порт 993 открыт).
Railway: `IMAP_PROXY_URL` задан → запрос уходит на Fly.io.

Единый механизм для обоих ящиков. Gmail API и XOAUTH2 — выброшены как усложнение.

### Что сделать Борису (деплой прокси)
- [ ] Установить flyctl: `winget install Superfly.flyctl`
- [ ] `fly auth login`
- [ ] `cd imap_proxy && fly launch --name ai-office-imap-proxy --region ams --no-deploy`
- [ ] `fly secrets set IMAP_PROXY_SECRET=<случайная-строка>`
- [ ] `fly deploy`
- [ ] Добавить в Railway: `IMAP_PROXY_URL=https://ai-office-imap-proxy.fly.dev`
- [ ] Добавить в Railway: `IMAP_PROXY_SECRET=<та же строка>`
