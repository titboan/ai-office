## Статус блокировок (2026-06-13)

### Telegram-каналы (Фаза 1) — заблокировано
my.telegram.org выдаёт generic «ERROR» при создании API-приложения.
Причина: Telegram спам-защита (новый IP / аккаунт).
Решение: подождать 1–2 ч, попробовать в инкогнито или с другого IP.
Нужно получить: `TELEGRAM_API_ID`, `TELEGRAM_API_HASH` → затем сгенерировать `TELETHON_SESSION`.

### Email через IMAP — заблокировано Railway
Код написан (tools/email_reader.py), переменные добавлены в Railway, но Railway
блокирует исходящий порт 993 (IMAP SSL) на Hobby/Free плане.
Подтверждено: локально (Windows) оба ящика работают — Gmail 33 641 писем, Яндекс 4 315.
На Railway — `[Errno 111] Connection refused` для обоих ящиков.

Дополнительный фактор: Google и Яндекс блокируют IMAP с IP облачных провайдеров
(Railway, HuggingFace и др.) — даже Pro план может не помочь.

Railway официально рекомендует HTTP API вместо прямых IMAP/SMTP соединений.

### Следующий шаг — Фаза 7: переход на HTTP API
Варианты (выбрать один):
- Gmail API (OAuth2, HTTPS) + Yandex Mail API — надёжно, не блокируется
- Railway Pro план — открывает SMTP, IMAP неизвестно, не гарантировано
- VPS (Hetzner/DigitalOcean) — гарантировано, но переезд
