# Ева — личный секретарь: Telegram-дайджест + дайджест почты

Статус: в работе
Дата: 2026-06-13

## Контекст

Ева написана (agents/eva.py): дайджест Telegram-каналов + email-дайджест двух ящиков (Gmail + Яндекс).
Код полностью готов. Единственный блокер: все бесплатные облачные платформы блокируют IMAP порт 993.

---

## Архитектура email (финальная)

```
Railway (eva) → HTTPS → IMAP-прокси (Timeweb VPS) → порт 993 → Gmail / Яндекс
```

Локально: `IMAP_PROXY_URL` пустой → прямое IMAP (работает).  
Railway: `IMAP_PROXY_URL` задан → запрос уходит на Timeweb.

Код готов. Ждёт деплоя прокси на Timeweb (~130 руб/мес).

---

## Ключевой вывод по инфраструктуре

**Все PaaS-платформы (Railway, Render, Fly.io) блокируют исходящий порт 993.**  
Попытки обходов:
- Gmail API (OAuth2) — работает только для Gmail, не для Яндекса
- XOAUTH2 IMAP — всё равно порт 993, не помогает
- Render.com — тоже блокирует порт 993
- Fly.io — требует карту, в РФ не работает
- Oracle Cloud Free — не принимает российские карты

Решение: **Timeweb VPS** (российский хостинг, ~130 руб/мес, порт 993 открыт).

## Фаза 7: Деплой IMAP-прокси на Timeweb (отложено, не срочно)

- [ ] Зарегистрироваться на timeweb.com → купить минимальный VPS (Ubuntu, ~130 руб/мес)
- [ ] Установить Docker: `curl -fsSL https://get.docker.com | sh`
- [ ] Скопировать `imap_proxy/` на сервер
- [ ] Запустить: `docker build -t imap-proxy . && docker run -d -p 8080:8080 -e IMAP_PROXY_SECRET=<секрет> imap-proxy`
- [ ] Добавить в Railway: `IMAP_PROXY_URL=http://<ip>:8080`, `IMAP_PROXY_SECRET=<секрет>`
- [ ] Удалить из Railway: `IMAP_PROXY_URL` (пока не задеплоен — там сейчас указан сломанный Render)
- [ ] Проверить: `/email_digest 1d` → дайджест обоих ящиков
