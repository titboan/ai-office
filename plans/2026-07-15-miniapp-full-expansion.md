# Мини-апп: вкладки, отчёты, каталог, формы вместо мастеров Макса

Статус: в работе

## Контекст

Заменяет `plans/2026-07-15-miniapp-reports-expansion.md` — скоуп расширен по итогам повторного
разбора всего проекта (не только Питера) глазами нового клиента.

Решения с Борисом (2026-07-15):
- Текстовые отчёты Питера **не убираем** из чата — короткое summary + кнопка «Открыть в
  дашборде» вместо дублирования логики (передумали с полного дублирования, которое обсуждали
  раньше — риск рассинхрона двух реализаций одних расчётов).
- Мини-апп получает вкладки/разделы вместо одного скролла.
- Делаем всё: SEO, поставки, каталог, формы вместо мастеров.
- Single-entry-через-Марту — **уже завершено** (коммит `d54556f`, 2026-07-14): polling выключен
  у всех ботов кроме Марты, `ROADMAP.md` обновлён отдельным коммитом. Это условие для форм ниже —
  все новые эндпоинты валидируют `X-Telegram-Init-Data` строго по `MARTA_BOT_TOKEN`
  (`main.py::_validate_init_data`), тот же принцип, что уже у `applyPrice`/`applyBid`/`setCost`.

Разбор находок — что переносим и почему:

| Функция сейчас (чат) | Проблема | Решение |
|---|---|---|
| `/seo_audit` — текст | Нет визуала, данные (`kw_top`) уже объявлены в API, не заполняются | Вкладка «Отчёты» + summary в чате |
| `/supply` — текст | Нет визуала вообще | Вкладка «Отчёты» + summary в чате |
| `/audit` — текст (SWOT) | Прозаический анализ, плохо ложится в виджет | **Не переносим**, остаётся как есть |
| `/products`, `/margin`, `/shop_kpi` | Текстовые простыни, данные уже структурированы в БД | Вкладка «Каталог» |
| `/map`, `/cost` (частично) | `key=value` синтаксис командной строки в чате | Форма в мини-аппе |
| `/add` (`cmd_add`) | Пошаговый Redis-wizard (название→категория→площадка) через чат | Форма (все поля сразу) |
| `/merge_products` | Постраничный inline-кнопочный пикер | Форма-подбор (dropdown/список) |
| `/add_shop` | Ввод API-токена текстом в чат | Форма (чувствительные поля, не логировать) |

Не в скоупе: `/analyze` (свободный вопрос — годится только чат), генеративный контент Элины
(`/write`/`/seo`/`/post` — результат для копирования, чат — правильный формат), тендеры Тины
(низкая частота, не приоритет), approve/reject-кнопки Макса/Питера (`/apply_prices`,
`/bid_adjust`, `/campaigns` — уже хороший паттерн, не трогаем).

Реализация — по фазам, каждая отдельным вызовом `implementer` (см. `CLAUDE.md` — оркестрация).
После каждой фазы: сам читаю diff, коммичу, только потом следующая фаза.
