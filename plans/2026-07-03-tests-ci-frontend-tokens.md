# Тесты + CI по бэкенду, и дизайн-токены + Tremor по фронтенду

Статус: завершён

## Контекст

По итогам анализа всего проекта (сессия 2026-07-03) выявлен один системный риск: во всём проекте (~20 000 строк) нет ни одного автотеста и нет CI. Минимум 3 продакшн-инцидента за последние 8 ретроспектив — это один и тот же класс бага: угадали формат/поле внешнего API (WB/Ozon) вместо проверки на реальных данных, регрессия ловилась только после жалобы пользователя. Отдельно на дашборде — рабочий, но визуально несистемный UI: 13 виджетов, в каждом продублирована карточка/цвета/пороги.

Цель этой фичи: закрыть тестами уже случившиеся баги (чтобы они не повторились незаметно), поставить минимальный CI, и убрать дублирование во фронтенде через общие токены/примитивы + Tremor для части графиков.

## Фаза 5 — фронтенд: Tremor для графиков — ПОПРОБОВАЛИ, ОТКАЗАЛИСЬ

Пилот на `MomChart` (самый простой график, один bar-ряд):
- [x] Добавили `@tremor/react`, обновили `tailwind.config.js` (content path + safelist для `violet`/`blue`)
- [x] Собрали и проверили в браузере (light + dark)

**Результат — два конкретных минуса, перевешивающих плюсы:**
1. **Бандл вырос с 160.86 KB до 284.47 KB gzip (+77%)** ради одного мигрированного графика — для мобильного Telegram Mini App, где важна скорость загрузки, это ощутимая цена за визуальный тюнинг.
2. **Цвет не применился** — столбцы отрендерились чёрными: Tailwind JIT не подхватил классы, которые Tremor генерирует динамически, несмотря на добавленный safelist. Официальный safelist Tremor кладёт в конфиг куда больше классов (bg/text/border/ring/fill/stroke × вся палитра × все оттенки) — рабочее решение требует скопировать их полный boilerplate, что не тривиально и хрупко при апдейтах Tremor.

**Решение: остались на Recharts.** Фаза 4 уже убрала реальную проблему (дублирование карточек/цветов/порогов) без новой зависимости и без риска для бандла. Tremor и его safelist-конфиг убраны из репозитория (`git diff` на `package.json`/`package-lock.json`/`tailwind.config.js` пуст).
