## Контекст

По итогам двух раундов анализа проекта (архитектура/security + продуктовая стратегия) часть
находок уже закрыта в предыдущих сессиях: тесты+CI, дизайн-токены дашборда, owner-only доступ,

приоритету, а не пытаться сделать всё разом.

Реализовывать по одному пункту за раз, начиная с Фазы 1.

  плюс жёсткий потолок безопасности.** `agents/max.py`: `clamp_wb_cpm()`/`clamp_ozon_bid()` —
  чистые функции, считают новое значение по проценту и зажимают в `[floor, config.WB_MAX_CPM_RUB]`

  от аномального `delta_pct`, не бизнес-правило). Используются в двух местах одинаково:

  и непосредственно перед записью в API в `_handle_bid_callback`/`_handle_ozbid_callback` —
  один источник правды, превью не может разойтись с тем, что реально применится. Тесты:
  `tests/test_bid_clamp.py` (8 тестов: процент вниз/вверх, пол, потолок для обоих маркетплейсов).
- [x] **Persistent company context — единообразно во всех system-промптах, не только в
  основном tool_use цикле.** Проверка показала: `config.COMPANY_CONTEXT` реально доходил
  только туда, где агент вызывал `self._effective_system` (базовый tool_use цикл
