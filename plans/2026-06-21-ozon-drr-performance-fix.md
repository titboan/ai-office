# План: исправление ДРР Ozon — два бага в Performance API

**Статус: завершён**

---

## Контекст

Дашборд показывал Ozon реклама = **109 923 ₽** (14 дней, июнь 7–21).  
Ozon в кабинете: **261 859 ₽** (21 день, июнь 1–21).  
Для сопоставимого 14-дневного периода ожидалось ~174 000 ₽ → дашборд занижал на ~37%.

Расхождение нельзя объяснить только Premium/Брендом (~25 958 ₽) — значит, баги в самом коде Performance API.

---

## Найденные баги (оба в `tools/marketplace.py`, `OzonPerformanceClient.get_ad_stats`)

### Баг 1 — ZIP: читали только первый файл из N

```python
# БЫЛО:
csv_text_batch = zf.read(zf.namelist()[0])  # только 1-й из N!
```

Ozon Performance API возвращает ZIP с **отдельным CSV на каждую кампанию**. При 10 кампаниях в батче → ZIP с 10 файлами → читался только первый. С 19 реальными кампаниями в 2 батчах захватывалось **2 из 19** (~10% расходов).

### Баг 2 — stat_date: ежедневные синки накапливали дубли

```python
# БЫЛО:
"stat_date": date_from,  # начало 7-дневного окна
```

Каждый ежедневный синк (03:00 UTC) использует `date_from = today − 7`. Значение меняется каждый день → новый `stat_date` → **новая строка вместо обновления существующей**. За 6 дней синков — 6 строк на кампанию, каждая хранит 7-дневный агрегат. Дашборд суммировал все → ~7× переоценка.

**Итоговый эффект обоих багов**: 10% захват × ~7× дублирование ≈ 65–70% от реального. Математически объясняет наблюдаемые 109 923 ₽.

---

## Решение

### Фикс 1 — ZIP: итерировать все файлы

```python
# СТАЛО:
for fname in zf.namelist():
    csv_text_file = zf.read(fname).decode("utf-8-sig", errors="replace")
    csv_texts.append(csv_text_file)
```

### Фикс 2 — stat_date: делить агрегат по дням

Вместо одной строки с `stat_date = date_from` и суммой за 7 дней — создаём отдельную строку на каждый день периода с `spend = total / days_count`. Ключ upsert `(campaign_id, stat_date)` уникален по дням → повторный синк перезаписывает, а не дублирует.

## Оставшийся известный разрыв

После фиксов Performance API покрывает **CPC + CPO ≈ 235 903 ₽** (июнь 1–21).  
Недостающие **25 958 ₽** (Premium 9 990 + Бренд 15 963 + Бонусы 2) не отдаются через Seller API — это известный разрыв (~10%), принят как норма.

Историческая попытка достать Premium/Бренд через `/v3/finance/transaction/list` (план `2026-06-15-drr-ozon-fin-adv.md`) не дала результата: угаданные имена `operation_type` не совпали с реальными (только логистика в ответах API). Документация Ozon блокирует автоматический фетч (403).
