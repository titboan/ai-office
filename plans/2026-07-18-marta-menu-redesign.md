# Редизайн меню Марты — от группировки по агентам к группировке по задачам

**Дата:** 2026-07-18
**Статус:** завершён

## Контекст

Аудит интерфейса бота Марты (сессия 2026-07-18) нашёл перегруженное меню: 45 команд в
нативном списке «/» Telegram (комфортно читается штук 10-15), 42 действия в инлайн-меню
`/menu`, при этом ~23 команды (включая денежно-чувствительные `/reprice`, `/cost`, `/map`,
`/camp`, `/add_shop`) недоступны через `/menu` вообще — только по памяти. Раздел
«🛒 Маркетплейсы» — 15 кнопок в 8 строк на одном экране. Есть 4 кнопки-«призрака» —
обработчики текста без соответствующих кнопок на клавиатуре (мёртвый код). Раздел
«⚙️ Офис» смешивает бизнес-настройку (поставки) с технической отладкой (статус очереди,
логи ошибок) — не годится, если ботом когда-нибудь будет пользоваться не только владелец.

Разделы `/menu` сейчас названы и сгруппированы по тому, какой агент отвечает за функцию
(«Аналитика — Питер», «Маркетплейсы — Макс»), а не по тому, что хочет сделать продавец —
из-за этого связанные по смыслу вещи (ДРР и ставки рекламы, анализ и синк возвратов)
разъезжаются по разным разделам.

## Фазы

### Фаза 1 — почистить мёртвый код кнопок [x]
- `agents/marta.py:494-591` (`_process_text`) — обработчики текста «⭐ Отзывы»,
  «🔄 Синхронизация», «📋 Статус», «📜 История» не соответствуют ни одной кнопке в
  `_main_keyboard()` (`agents/marta.py:160-164`, там только 4 кнопки: Отчёт/Дашборд/Меню/
  Помощь). Решить: либо вернуть эти 4 пункта на клавиатуру (если они полезны как быстрый
  доступ), либо убрать неработающие обработчики как мёртвый код.

### Фаза 2 — перегруппировать `/menu` по задачам, а не по агентам [x]
Рабочее предложение группировки (можно скорректировать перед реализацией):
- **💰 Цены и реклама** — маржа, применить цены, ставки рекламы, ДРР/ROAS, кампании/акции
  Ozon (сейчас разбросано между «Аналитика» и «Маркетплейсы»)
- **⭐ Отзывы и вопросы** — обработать отзывы, вопросы, модерация
- **📊 Отчёты** — отчёт продаж, аудит, воронка, ABC, возвраты, SEO-аудит, свободный вопрос
- **🔄 Данные** — полный синк, финотчёт, реклама, воронка/возвраты/карточки (данные)
- **🏪 Магазины и каталог** — магазины, товары, KPI магазина, статус данных, позиции ключей
- **✍️ Контент и планы** — текст/пост/SEO (Элина), исследование (Каспер), напоминания
  (Алекс), тендеры (Тина)
- **⚙️ Служебное** — только для владельца: статус очереди, история задач, логи ошибок,
  настройки поставок
- Реализовать как правку `_MARTA_MENU_SECTIONS`/`_MARTA_MENU_KEYBOARD`
  (`agents/marta.py:1505-1686`) — без изменения самих действий (`mmenu_run:*` остаются
  теми же), только перегруппировка кнопок по разделам и, где нужно, разбивка раздела
  «Маркетплейсы» (сейчас 15 кнопок на одном экране) на 2-3 более мелких.

### Фаза 3 — сократить нативный список команд «/», не теряя доступности [x]
- `agents/marta.py:1838-1888` (`_register_bot_commands`) — оставить в нативном
  автокомплите Telegram только самые частые ~12-15 команд (отчёт, аудит, ДРР, отзывы,
  вопросы, синк, дашборд, помощь, меню и т.п.), убрать редкие/служебные из этого списка.
- Все команды, которых сейчас нет в `/menu` (`/reprice`, `/cost`, `/map`, `/camp`,
  `/add_shop`, `/set_performance`, `/reset_checked`, `/reset_orders`, `/sync_sku`,
  `/sync_promotions`, `/set`, `/code`, `/plan`, `/testpush`, `/roadmap`, `/tender`,
  `/dashboard`), добавить кнопками в соответствующие разделы `/menu` из Фазы 2 — чтобы
  сокращение нативного списка не превратило их в полностью недоступные без знания
  команды наизусть.

### Фаза 4 — разделить бизнес-настройку и техническую диагностику [x]
- `agents/marta.py:1670-1684` (раздел `office`) — вынести «📦 Настройки поставок» в
  раздел «Магазины и каталог» (Фаза 2), оставить в «⚙️ Служебное» только
  технико-диагностические пункты (статус очереди, история задач, логи ошибок) — этот
  раздел явно подписать как для владельца/разработчика, не для бизнес-пользователя.
