# Запуск агента Тина (44-ФЗ)

## Контекст

Код агента Тина написан и влит в main.py, но агент не активен: нет Telegram-бота и переменных в Railway.
Этот план — чек-лист для Бориса: от создания бота до первого дайджеста.

## Фазы

- [ ] **Фаза 1**: Создать Telegram-бота
  - Telegram → @BotFather → `/newbot`
  - Имя: `Тина — Тендеры`, username: `tina_tenders_bot` (или любой свободный)
  - Сохранить токен `7xxxxxxxxx:AAF...` — это `TINA_BOT_TOKEN`

- [ ] **Фаза 2**: Добавить переменные в Railway
  - railway.app → проект ai-office → Variables
  - `TINA_BOT_TOKEN = <токен из фазы 1>`
  - `TAVILY_API_KEY` уже есть (Каспер), добавлять не нужно

- [ ] **Фаза 3**: Зарегистрироваться на ГосПлан API *(до 01.08.2026 бесплатно)*
  - Перейти на https://v2.gosplan.info → Регистрация → получить ключ
  - Добавить в Railway: `GOSPLAN_API_KEY = <ключ>`
  - Если пропустить — агент работает без ключа до 01.08.2026

- [ ] **Фаза 4**: Деплой
  - Claude Code пушит ветку `claude/tender-agent-launch-plan-lsmz79`
  - Борис мёрджит PR в main
  - Railway автоматически деплоит, в логах: `✅ TinaAgent started`

- [ ] **Фаза 5**: Проверка
  - Открыть бота Тины в Telegram
  - Отправить `/tenders матрасы` → ждать 1-2 минуты
  - Должен прийти отчёт с рекомендациями УЧАСТВОВАТЬ / ПРОПУСТИТЬ / АНАЛИЗИРОВАТЬ
  - Проверить `/tenders_report` — сохранённые тендеры из БД

## Команды Тины

- `/tenders [ключевое слово]` — поиск и анализ тендеров (1-2 мин)
- `/tenders_report` — сохранённые тендеры из БД с маржой
- `/tender <lot_id>` — полный анализ конкретного тендера

Дайджест: ежедневно **08:00 МСК** автоматически.
