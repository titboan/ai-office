## Команда

```powershell
cd "C:\Users\Дом\ai-office"

gh pr create `
  --title "fix: не транскрибировать голосовые в группе без reply-to-bot у Макса" `
  --body "## Проблема

В групповых чатах Макс транскрибировал через Groq Whisper все голосовые сообщения — даже не адресованные боту. Проверка триггеров происходила ПОСЛЕ транскрипции:

- лишние вызовы Groq API (расходы)
- индикатор «typing...» у бота на нерелевантные сообщения

## Решение

В \`agents/max.py\` (\`handle_voice\`) добавлен pre-check до вызова \`super()\`:

- **В группе:** транскрипция только если голосовое — reply на сообщение бота
- **В личке:** поведение не изменилось
- Все остальные голосовые в группе — \`return None\` без Groq вызова

## Trade-off

Голосовой триггер «макс, ...» в группе без reply-to-bot больше не работает. Приемлемо — в группе можно написать «макс ...» текстом или ответить на сообщение бота голосом.

## Проверка

1. Голосовое в личку Максу → транскрибируется, бот отвечает
2. Голосовое в группу без reply → нет реакции, нет «typing...»
3. Reply на сообщение Макса в группе голосом → транскрибируется, бот отвечает"
```
