# Красивый визуал алерта остатков + Марта понимает "Остальные"

Статус: завершён

## Доп. улучшение (02.07)
- Скорость продаж в `_check_stock_alerts` считается только по дням, когда товар был в наличии
  (JOIN с `stock_history_daily`), а не по всему окну — дефицит больше не занижает рекомендацию заказа
- Окно для скорости продаж теперь = порог алерта (доставка+запас), но не короче 7 дней,
  вместо жёстко зашитых 14 дней

## Проблемы
1. Сообщение обрезается: только 10 позиций + "...и ещё N позиций"
2. Нет данных о товарах в пути (in_transit) — не учитываются в рекомендации
3. Визуал "сырой" — нет секций, нет объяснения формулы
4. Марта при "Остальные покажи" не понимает контекст и просит уточнения

## Решение

### Фаза 1 — Улучшить _check_stock_alerts [x]
- SQL: добавить LEFT JOIN marketplace_in_transit
- Формула: qty_to_order = max(0, (lead+safety)*vel - in_transit)
- Показывать "🚚 В пути: N шт" в сообщении
- Секции: "🔴 Нет остатков" и "⚠️ Заканчивается"
- Убрать лимит 10 — слать чанками по 10 через Мартин бот
- При >10 позиций: сохранить overflow в Redis (key: stock_overflow:{chat_id}, ttl: 24h)

### Фаза 2 — Марта обрабатывает "Остальные покажи" [x]
- В _process_text добавить детект паттернов "остальн/покажи все/следующ/ещё позиц"
- Если есть stock_overflow:{chat_id} в Redis → показать оставшиеся позиции
- Очистить Redis после показа
