## Контекст проекта

ИИ-офис: команда из 9 специализированных агентов на Claude API, управляемых через Telegram.
Один Python процесс на Railway. Репозиторий: `titboan/ai-office` (приватный).

- **Деплой:** Railway (один сервис, все агенты в одном процессе)
- **Статус:** 🟢 Продакшн
- **ОС разработчика:** Windows / PowerShell — использовать `Select-String` вместо `grep`

---

## Принципы (нарушать только с явным обоснованием)

1. **Один Python процесс** — никаких микросервисов
2. **Postgres — источник истины.** Задачи, состояния, результаты — только там
3. **Notion — UI и knowledge base.** Не транзакционный backend
4. **Агенты изолированы.** Никаких прямых вызовов между агентами — только через очередь
5. **Человек подтверждает опасные действия** — деплой, PR, внешние API
6. **Простота важнее фреймворков.** Никаких LangGraph, CrewAI, Celery, DSPy
7. **Эволюция, не переписывание.** Минимальный фикс поверх работающего

## Рабочий процесс разработки

1. Claude в чате (claude.ai) анализирует задачу
2. Claude Code в расширении VS Code применяет изменения
3. Борис деплоит на Railway и сообщает результат

**Claude Code = исполнитель. Claude в чате = архитектор.**

### Правила git
- `git add` только поимённо — никогда `git add .` или `git add -A`
- Коммит после каждой завершённой задачи, осмысленное сообщение
- Перед push проверить `git log origin/main..main --oneline`

### Правила безопасности
- `.env` не читать целиком — для конкретной переменной использовать `grep "^VAR=" .env`
- Не деплоить без явной команды от Бориса
- `Conflict` при деплое — норма, само проходит 30-60 сек

## Планы (plans/)

В начале каждой сессии проверь папку plans/ — если есть файлы
со строкой "Статус: в работе", покажи их список и спроси продолжать.

Формат файла плана:
- Имя: YYYY-MM-DD-название-задачи.md
- Фазы с чек-боксами: [ ] или [x]
- Строка "Статус: в работе / завершён"

После завершения каждой фазы:
- Отметить выполненные шаги [x]
- Если все фазы [x] — изменить статус на "завершён"

Когда пользователь говорит "составь план" или "сохрани план":
- Создать файл в plans/ с правильным именем
- Разбить на фазы с чек-боксами
- Поставить "Статус: в работе"
