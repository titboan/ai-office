# Ретроспектива: переход на Telegram HTML + голос у Макса + ai-clone
**Дата:** 2026-06-12

---

## 1. Какая задача была

Три темы в одной сессии:
- Перевести форматирование с Markdown на Telegram HTML (`<b>`, `<i>`, `<code>`) + статус в pinned message
- Починить обработку голосовых у Макса в групповых чатах
- Ввести систему самообучения агента: `ai-clone/feedback/`, переработать CLAUDE.md, добавить deny-правила в settings.json

## 2. Как решали

**Telegram HTML:**
- Глобальная замена `*текст*` → `<b>текст</b>` в max.py и других агентах
- Добавили `parse_mode=HTML` там, где раньше был Markdown или ничего
- Pinned message: Марта пишет статус один раз при старте и обновляет его, а не слёт каждую минуту (убрали polling каждую минуту → только по запросу)

**Голосовые у Макса:**
- Баг: в группе транскрипция отправлялась всем, даже если бот не был адресатом
- Фикс: проверяем `reply_to_message` — транскрибируем только если реплай адресован боту
- `override_text` — передали transcribed text через явный параметр в `handle_message`, а не через `update.message.text`
- Исключения в `_notify_pending` теперь перехватываются, чтобы не блокировать polling loop

**ai-clone + CLAUDE.md:**
- Создали `ai-clone/feedback/` — место для накопленных правил работы с Борисом
- Переработали CLAUDE.md: убрали дубли, добавили порядок чтения в начале сессии, ссылки на skills
- Добавили deny-правила в `.claude/settings.json` (опасные операции требуют явного подтверждения)
- `plans/` — добавили правило: перед стартом задачи создавать план

## 3. Решили — да

Все три линии закрыты. Планы помечены `завершён`.

## 4. Что можно было лучше

- Переход на HTML лучше делать одним большим коммитом на весь проект, а не по файлам — тогда не будет ситуации, когда часть агентов уже HTML, часть ещё Markdown.
- `_notify_pending` без try/except — это была бомба замедленного действия. Стоило добавить при первичном написании.

## 5. Что узнал нового о проекте

- PTB polling loop падает целиком если в `_notify_pending` бросается необработанное исключение — критичный путь, требует обёртки.
- `override_text` — правильный способ передать уже обработанный текст в `handle_message` вместо мутации `update.message.text`.
- Порядок чтения в CLAUDE.md теперь явный: сначала `ai-clone/feedback/`, потом `plans/`, потом зона задачи.

## 6. Нарушения правил

Нет. deny-правила в settings.json добавлены именно потому, что в прошлом агент мог выполнить опасную операцию без подтверждения — теперь это зафиксировано на уровне конфига.
