# Рефлексия: Переход с HTML на MarkdownV2

**Дата:** 2026-06-14
**Задача:** Мигрировать все агенты с `parse_mode="HTML"` на `parse_mode="MarkdownV2"` в связи с обновлением Telegram — полноценная поддержка Markdown (заголовки, таблицы, HR).

---

## 1. Какая задача была

Борис показал скриншот объявления Telegram о полноценной поддержке Markdown и скриншот из чата где отчёт Питера выглядел как сырой markdown — виден символ `--` между секциями. Хотел чтобы отчёты рендерились красиво.

## 2. Как решали

1. **Диагностика** — по скриншоту определили что `--` — это Claude генерирует markdown-разделители несмотря на HTML-промпт.
2. **Быстрый фикс (сначала)** — добавили `clean_agent_output()` и запрет `--` в промпте. Но потом Борис показал новый Telegram Markdown и захотел полный переход.
3. **Планирование** — составили план с 4 фазами, Борис одобрил.
4. **Реализация**:
   - `utils/tg_format.py`: `MARKDOWN_FORMAT_RULES`, `escape_mdv2()`, `strip_mdv2()`
   - `base_agent.py`: `_notify_user` + `handle_message` → MarkdownV2 с fallback
   - 7 системных промптов агентов: HTML-правила → MarkdownV2
   - `marta.py`: 25 hardcoded HTML-строк → MarkdownV2 с `escape_mdv2()` для динамических данных

## 3. Решили — да / нет / частично

**Частично.** Код написан и запушен, но:
- Не протестировано в живом боте (Railway не задеплоен на момент сессии)
- MarkdownV2 escaping в динамических строках может иметь edge-cases (пейлоады задач с `(`, `.`, `-`)
- Неизвестно точно какой parse_mode поддерживает новые фичи Telegram — возможно `"MarkdownV2"` уже поддерживает таблицы и заголовки, или нужен новый режим

## 4. Что можно было лучше

- **Сначала проверить одного агента (Питера)** — вместо глобальной миграции всех 9 агентов сразу. Если бы MarkdownV2 сломался, было бы проще откатить.
- **Уточнить parse_mode** — перед началом надо было разобраться, добавил ли Telegram новый `parse_mode` или расширил существующий `"MarkdownV2"`. Решили использовать `"MarkdownV2"` по умолчанию как наиболее безопасный вариант.
- **Обнаружил ошибку по ходу** — написал `**bold**` в системных промптах (CommonMark), а MarkdownV2 использует `*bold*`. Поймал до пуша.

## 5. Что узнал нового о проекте / пользователе

- Борис следит за обновлениями Telegram и хочет использовать новые возможности быстро
- В marta.py очень много hardcoded HTML-строк (статус, история, проекты, help) — технический долг
- `escape_mdv2()` нужен везде где в MarkdownV2-сообщении появляются динамические данные из БД (пейлоады, имена агентов с цифрами/дефисами, даты)

## 6. Нарушил ли агент правило молча?

Нет очевидных нарушений. Но стоит добавить в проверку: при работе с MarkdownV2 всегда оборачивать динамические данные в `escape_mdv2()` — иначе Railway упадёт с первым пейлоадом содержащим `.` или `(`.
