# 2026-06-24 — Единая точка входа через Марту: прокси-команды и QA-фикс

## Задача

Борис устал переключаться между 8 ботами — нет единой истории. Нужна единственная точка входа через чат Марты. Параллельно: быстрое меню с описаниями, кнопка «Назад», и QA всех сломанных команд.

## Как решали

**Фаза 1 — прокси-команды (предыдущая сессия, перенесено из контекста):**
- Добавили ~40 proxy-команд на App Марты: все команды всех агентов через `_proxy_cmd()` → `enqueue_task(from_agent="marta")` → результат идёт через MARTA_BOT_TOKEN (уже работал в `_ORIGIN_TOKENS`).
- Добавили инлайн-меню `/menu` с 7 разделами и кнопкой «Назад» через `edit_message_text` (не новое сообщение — нет скролла).

**QA-диагностика (эта сессия):**
- Борис прислал скриншот: `/shops` через Марту → "нет инструментов для получения информации".
- Задача: найти все аналогичные баги.
- Диагностика: `handle_task` у Макса (строка 244) делал `think(task, chat_id=0)` — обычный вызов Claude без инструментов и без данных для ЛЮБОЙ не-JSON задачи из очереди.
- Claude получал описательный русский текст ("покажи список магазинов"), MAX_SYSTEM (промпт про отзывы), никаких tools → отвечал честно: "нет инструментов".

**Список сломанных (все команды Макса через Марту):**
- Чтение: `/shops`, `/products`, `/data_status`, `/shop_kpi`, `/seo_check`, `/bid_adjust`, `/margin`
- Синхронизация: `/sync`, `/sync_fin`, `/sync_adv`, `/sync_funnel`, `/sync_returns`, `/sync_cards`
- Per-item с кнопками: `/questions`, `/pending` (Phase 2)

**Фикс:**
1. Marta: payload прокси → `__keyword__` префиксы вместо описательного текста
2. Max.`handle_task`: `if task.startswith("__") → _dispatch_queue_task(task, _current_chat_id)`
3. `_dispatch_queue_task`: dispatch table → вызов реальных внутренних методов
4. Новые helper'ы `_X_text(chat_id) → str`: shops, shop_kpi, seo_check, data_status, bid_adjust
5. Рефактор cmd-методов → вызывают те же helper'ы (DRY)

## Решили: да

Все простые чтение/синк команды теперь работают через очередь. `/questions` и `/pending` дают информативный текст про Phase 2.

## Что можно было лучше

- Баг с `think(chat_id=0)` существовал с момента написания первых прокси-команд в предыдущей сессии — можно было поймать тогда же, если сразу тестировать каждую группу команд.
- Диагностику начал с чтения marta.py, хотя root cause был в max.py — лучше сразу читать оба файла параллельно.

## Что узнал нового

- Max.`handle_task` не имеет dispatch — всё через `think()`. Питер имеет keyword-матчинг. Разные архитектуры у разных агентов.
- `_current_chat_id` устанавливается воркером до `handle_task` (base_agent.py:733) — его безопасно использовать внутри.
- У Макса уже были `_margin_check_text(chat_id)` и все sync-методы с `(chat_id)` сигнатурой — архитектура была готова, не хватало только dispatch.

## Нарушение правил

Нет.
