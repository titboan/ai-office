# Формат в промпте обязан совпадать с parse_mode отправки

**Rule:** Если системный промпт агента диктует формат (MarkdownV2 или HTML), то **все** пути отправки вывода Клода (`reply_text`, `send_message`, кастомные `_send_*`) должны использовать тот же `parse_mode`. При миграции формата — пройтись grep'ом по всем `parse_mode=` и `reply_text(`/`send_message(`, а не только по промптам.

**Why:** Миграция на MarkdownV2 (коммит `23c58f3`) обновила системные промпты, но оставила `parse_mode="HTML"` (или вообще без parse_mode) в командах-обёртках peter/elina/alex/kasper/kevin. Telegram не рендерил `*жирный*` → пользователь видел сырые звёздочки. Баг проявился у Питера на проде.

**How to apply:** Срабатывает при любой работе с форматированием Telegram-сообщений. Проверочный чеклист для каждого агента:
1. Что говорит `*_SYSTEM` промпт — MarkdownV2 или HTML?
2. Каждый `reply_text`/`send_message` с **выводом Клода** (`answer`, `result`, `think()`) использует тот же `parse_mode`?
3. Есть ли fallback `except → strip_mdv2(...)` (для MD) — иначе при неэкранированных спецсимволах пользователь увидит сырой текст со звёздочками.
4. Хардкод-отчёты из f-строк (как у max) — отдельный случай: их формат задаётся кодом, parse_mode под них.
5. Notion/GitHub-ссылки, дописываемые в ответ, — в формате текущего parse_mode (`[текст](url)` для MD, `<a href>` для HTML). Помни: `_clean_output` режет HTML-теги.
