# Правила планирования задач

**Rule:** Любая задача с 2+ файлами, новым агентом/таблицей/эндпоинтом, или когда сам не уверен в объёме — требует плана. Сомневаешься — создай план.

**Why:** Борис хочет видеть scope и согласовывать подход до реализации. Без плана сюрприз в конце сессии. Планы — часть проекта, должны лежать в репо.

**How to apply:**

**Где создавать:** `plans/YYYY-MM-DD-название-задачи.md` в корне репо. Никогда не в `.claude/plans/` или системных путях.

**Структура:**
```
# Название

Статус: в работе

## Контекст
<одна строка — зачем этот план>

## Фазы
- [ ] Фаза 1: ...
- [ ] Фаза 2: ...
```

**После завершения всех фаз:** поменять `Статус: в работе` → `Статус: завершён`.

**Типичные ошибки:**
- Создать план без чек-боксов и строки `Статус:` — неправильно
- Создать план вне `plans/` репо — неправильно
- Закрыть план без резюме в чат (см. `plan-completion-summary.md`) — неправильно
