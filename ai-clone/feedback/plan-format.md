# Формат планов

**Rule:** Планы всегда создавать в `plans/YYYY-MM-DD-название-задачи.md` в корне проекта. Никогда не в системных путях `/root/.claude/plans/` или других местах.

**Обязательная структура:**
```
# Название плана

Статус: в работе

## Контекст
<одна строка — зачем этот план>

## Фазы

- [ ] **Фаза 1**: ...
- [ ] **Фаза 2**: ...

После завершения всех фаз — поменять Статус на `завершён`.
```

**Why:** Борис смотрит на статус планов в начале каждой сессии. Если план не в `plans/` и без `Статус:` — он его не найдёт.

**Типичные ошибки:**
- Создать план как narrative how-to без чек-боксов — неправильно
- Создать план в `/root/.claude/plans/` или любом другом месте — неправильно
- Не добавить строку `Статус: в работе` — неправильно
