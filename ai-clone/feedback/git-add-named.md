# git add только поимённо

**Rule:** Никогда не использовать `git add .` или `git add -A` — только `git add файл1 файл2`.

**Why:** Риск случайно закоммитить `.env`, временные файлы, или незаконченные изменения в других файлах.

**How to apply:** Перед любым `git add` явно перечислить только те файлы, которые относятся к текущей задаче.
