Перед реализацией любого сетевого взаимодействия — сначала убедись что платформа его допускает.

**Почему:** В сессии 2026-06-14 написали Gmail API, XOAUTH2 и IMAP-прокси на Render — и только после деплоя выяснили что Render тоже блокирует порт 993 (как и Railway). Потратили несколько часов на код который не мог заработать.

**Как применять:** Перед реализацией нового сетевого пути — один быстрый тест:
- Нужен IMAP (993)? → `curl -v telnet://imap.gmail.com:993` с целевой платформы
- Нужен нестандартный порт? → Проверить документацию платформы по сетевым ограничениям
- Платформа облачная (PaaS)? → По умолчанию считать что порты кроме 80/443 заблокированы, пока не доказано обратное

Правило: **инфраструктура раньше кода**.
