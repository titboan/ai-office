# Правило: из plan mode выходить с одним чётким следующим шагом

## Проблема

В сессии 2026-06-21 три раза выходил в ExitPlanMode с планом у которого Фаза 0 требовала действий пользователя (проверить логи). Каждый раз получал уточняющий вопрос и возвращался в план.

Причина: план был перегружен сценариями А/Б/В "на случай если", вместо того чтобы зафиксировать ближайший конкретный шаг.

## Правило

Если следующий шаг плана — **эмпирические данные** (логи, скриншот, запрос к БД), и без них код написать невозможно:

1. Выйти из plan mode с ОДНИМ чётким действием для пользователя:
   > "Нужно запустить `/sync_adv` и прислать строку из логов с `[xxx]`"

2. Не описывать все возможные сценарии заранее — написать их ПОСЛЕ получения данных.

3. Не затягивать plan mode в ожидании "полной картины". Данных достаточно чтобы двигаться.

## Когда применять

Любой план где Фаза 0 = "получить данные от пользователя или из внешней системы". Выйти из plan mode → получить данные → вернуться к реализации. Не держать пользователя в режиме ожидания внутри plan mode.
