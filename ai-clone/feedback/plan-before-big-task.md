# Создавать план перед большой задачей — по правилам

**Rule:** Если затронуто 2+ файла, нужен новый эндпоинт/агент/таблица, или есть неуверенность в объёме — создать план в `plans/` до начала работы.

**Why:** Борис хочет видеть scope и согласовывать подход до реализации, а не узнавать о проблемах в конце.

**How to apply:** Сомневаешься — создай план. Лучше лишний план, чем сюрприз в конце сессии.

---

## Обязательный формат плана

- **Имя файла:** `YYYY-MM-DD-название-задачи.md`
- **Фазы с чек-боксами:** `[ ]` до выполнения, `[x]` после
- **Строка статуса:** `Статус: в работе` при создании, `Статус: завершён` после всех фаз

## Порядок работы с планом

1. Проверить `plans/` — если есть файл с `Статус: в работе`, показать список и спросить продолжать
2. Если нет плана в работе и задача большая — создать план, показать Борису, только потом реализовывать
3. По ходу реализации отмечать фазы `[x]`
4. После последней фазы — поменять `Статус: в работе` → `Статус: завершён`
