# Правило: не использовать данные одного МП как прокси для другого

## Контекст

Питер при анализе Ozon-вопросов несколько раз использовал WB-регионы продаж как «оценку»
для Ozon-складов — когда данных по Ozon не было. Борис явно сказал: так делать не нужно.

## Правило

**Маркетплейсы анализируются независимо. Никаких кросс-МП подмен.**

- Данные WB → только для WB-вопросов
- Данные Ozon → только для Ozon-вопросов
- Если данных по одному МП нет — пиши «данных нет», и не заменяй данными другого МП
- Сравнение WB vs Ozon — **только если пользователь явно попросил сравнить**

## Где зафиксировано в коде

- `PETER_SYSTEM` в `agents/peter.py` — раздел «ПРАВИЛО — маркетплейсы анализируются независимо»
- `handle_task` промпт в `agents/peter.py` — инструкция по `ozon_warehouses`
- `cmd_supply` промпт в `agents/peter.py` — правило по `ozon_warehouses`
